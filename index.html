HTML

  • HTML stands for Hyper Text Markup Language
  • The head element contains metadata: information about the webpage.
  • The body element represents the visible content shown to the user.
  • An attirbute defines additional info about an element.
  • In semantic html elements, the name of the element describes its content and the role it plays on a webpage.
  • It is good practice to use semantic html as it makes the webpage more accessible to users. ex. users who use screen readers.
  • Element and tag are often used interchangeably but they are different: Element refers to the full element from opening to closing tag, whereas tags refer to only what appears in the angle brackets.

CSS

  • CSS stands for Cascading Style Sheets.
  • Inline CSS is where CSS is coded induvidually into elements of html code.
  • Internal CSS is when CSS code is placed in the head elements metadata and effects multiple elements at once.
  • External CSS is when CSS code is in a seperate file and is added via a link element placed in the head head elements metadata.
  • A margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • 2 or more elements that share the same styling can be grouped together via a comma.

Git

  • Git commands should always start with git.
  • Git is a free open source version control system that allows devs to collab on a project w/o overwriting each others work.
  • Git allows us to revert to previous versions of code if a problem or "bug" is discovered.
  • Git status: checks what branch we are currently on.
  • Git checkout is how we move from branch to branch.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • Git add -A is how we stage a commit followed by git commit -m "A Message" to provide details about the staged commit.
  • Git pull origin main is how we keep our local files up to date and synced with our repo.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • In JS to declare a variable we use the var keyword; we give the variable a unique name which is how we reference the info stored in the variable in our code.
  • We use assignment operators to assign specific data we want stored in our variable.
  • Console refers to a test enviroment that devs used to check their code.
  • An array is a single variable that is used to hold a group of data that is related in some way.
  • String data is stored in quotation marks; number and boolean values do not need to be in quotations.
  • Index or indices are a number that identifies the location of data item within an array starting at 0 and progressing incrementally by 1.
  • Functions are a set of instructions that tell a PC how to perform a task.
